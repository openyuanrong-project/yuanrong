# 避免过度并发

相比普通函数调用，并发或者分布式调用需要更多的工作负载。并发调用过于细粒度的任务时反而比普通函数调用更低效！

为了避免上述问题，需谨慎设计函数并规划并发任务。如果要并发调用一个细粒度的任务，可以使用批量处理，让每一次调用处理更多有效工作。

## 代码示例

```python
# demo.py
import yr
import time
import itertools

yr.init()

numbers = list(range(10000))


# ------------------- Ordinary function call ---------------------------
def double(number):
    time.sleep(0.00001)
    return number * 2


start_time = time.time()
serial_doubled_numbers = [double(number) for number in numbers]
end_time = time.time()
print(f"Ordinary function call takes {end_time - start_time} seconds")
# Ordinary function call takes 0.7116954326629639 seconds
# ------------------- Ordinary function call ---------------------------

# ------------------- Parallelizing tasks ---------------------------

@yr.invoke
def remote_double(number):
    return double(number)


start_time = time.time()
doubled_number_refs = [remote_double.invoke(number) for number in numbers]
parallel_doubled_numbers = yr.get(doubled_number_refs)
end_time = time.time()
print(f"Parallelizing tasks takes {end_time - start_time} seconds")
# Parallelizing tasks takes 7.060078382492065 seconds

# ------------------- Parallelizing tasks ---------------------------

# ------------------- Parallelizing tasks with batching ---------------------------

@yr.invoke
def remote_double_batch(numbers):
    return [double(number) for number in numbers]


BATCH_SIZE = 1000
start_time = time.time()
doubled_batch_refs = []
for i in range(0, len(numbers), BATCH_SIZE):
    batch = numbers[i: i  BATCH_SIZE]
    doubled_batch_refs.append(remote_double_batch.invoke(batch))
parallel_doubled_numbers_with_batching = list(
    itertools.chain(*yr.get(doubled_batch_refs))
)
end_time = time.time()
print(f"Parallelizing tasks with batching takes {end_time - start_time} seconds")
# Parallelizing tasks with batching takes 0.0927739143371582 seconds

# ------------------- Parallelizing tasks with batching ---------------------------

yr.finalize()
```

上述例子展示了并发调度的效率不如普通调用的效率。但是通过批量处理，使并发中的每一次调用处理了更多的任务，从而实现了预期的处理效率。
